# Reelix AI

An AI-powered short-form reel generator. Upload images, add a voice description, and get a generated video reel with AI narration.

**Live demo:** [reelix-ai-mue8.onrender.com](https://reelix-ai-mue8.onrender.com)

---

## What it does

- Upload multiple images (JPG, PNG, WebP)
- Enter a text description — converted to speech using Google TTS
- A video reel is generated by stitching your images with the AI voiceover
- View all generated reels in the gallery

The longer your description, the more images get included in the final reel.

---

## Tech stack

- **Backend:** Flask (Python)
- **Video processing:** FFmpeg
- **Text to speech:** gTTS (Google Text-to-Speech)
- **Cloud storage:** Cloudinary (images + videos)
- **Deployment:** Render (free tier)

## How it works

1. User uploads images and enters text on `/create`
2. Images are uploaded to Cloudinary under `reelix/{job_id}/`
3. A job folder is created in `user_uploads/{job_id}/` with `desc.txt`
4. A background thread runs `generate_process.py` every 4 seconds
5. The worker finds new jobs, generates audio via gTTS, downloads images from Cloudinary, runs FFmpeg to stitch everything into a `.mp4`
6. The finished video is uploaded to Cloudinary under `reelix/reels/`
7. The gallery fetches all videos from Cloudinary and displays them

## Running locally

**1. Clone the repo**
```bash
git clone https://github.com/pranavgarg13-star/reelix-ai.git
cd reelix-ai
```

**2. Create and activate a virtual environment**
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Install FFmpeg**

Download from [ffmpeg.org](https://ffmpeg.org/download.html) and add it to your system PATH.

**5. Set up environment variables**

Create a `.env` file in the project root:
```
CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
```

Get these from your [Cloudinary dashboard](https://cloudinary.com).

**6. Run the app**
```bash
python main.py
```

Visit `http://localhost:5000`

## Deploying to Render

1. Push your code to GitHub
2. Go to [render.com](https://render.com) and create a new **Web Service**
3. Connect your GitHub repo
4. Set the following:
   - **Build command:** `pip install -r requirements.txt`
   - **Start command:** `gunicorn main:app`
5. Add environment variables in the Render dashboard:
   - `CLOUDINARY_CLOUD_NAME`
   - `CLOUDINARY_API_KEY`
   - `CLOUDINARY_API_SECRET`
6. Deploy

> Note: Render's free tier has an ephemeral filesystem. All persistent storage uses Cloudinary. The free tier also has a 512MB memory limit — the FFmpeg settings are tuned to stay within this.

---

## Known limitations

- Free tier on Render spins down after inactivity — first request may be slow
- Reel length is determined by audio length. Longer descriptions = more images included
- No user authentication — gallery is public and shared across all users
- File uploads are limited to 16MB
